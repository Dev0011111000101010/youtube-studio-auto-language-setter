# YouTube Studio Auto Language Setter

## Назначение

Playwright-демон, который мониторит YouTube Studio канала, находит видео со статусом **Private** и автоматически выставляет им язык **Russian**.

---

## Как работает

### Запуск
```
PYTHONIOENCODING=utf-8 .venv/Scripts/python main.py
```

> На Windows (cmd/PowerShell) используй:
> ```
> set PYTHONIOENCODING=utf-8 && .venv\Scripts\python main.py
> ```

### Алгоритм (loop каждые 5 минут)

```
1. Открыть браузер (persistent context → browser_profile/)
2. Перейти на страницу видео YouTube Studio
3. LOOP каждые 5 минут:
   a. Обновить страницу
   b. Собрать все видео со статусом "Private"
   c. Сравнить с state.json (уже обработанные ID)
   d. Для каждого НОВОГО Private видео:
      - Открыть страницу Details видео
      - Найти поле Language → выставить Russian
      - Кликнуть Save
      - ДОЖДАТЬСЯ ПОДТВЕРЖДЕНИЯ СОХРАНЕНИЯ (читать HTML/DOM)
        ↳ Логика подтверждения: определяется по реальной странице
        ↳ Может быть: исчезновение кнопки Save, появление toast-уведомления,
          изменение URL, изменение состояния элементов — уточняется при разработке
      - Добавить НАЗВАНИЕ видео в in-memory set (processed_titles)
   e. Вернуться на страницу со списком видео
   f. sleep(300 секунд)
```

---

## Аутентификация

Используется `Playwright launch_persistent_context` с папкой `./browser_profile/`.

- При **первом запуске**: браузер открывается, пользователь логинится в Google → сессия сохраняется в `browser_profile/`
- При **последующих запусках**: браузер стартует уже залогиненным

Папка `browser_profile/` — в `.gitignore` (содержит персональную сессию).

## Важные решения по логике

### Верификация сохранения (Save confirmation)

После клика Save **нельзя** просто перейти дальше. Нужно убедиться, что страница действительно зафиксировала изменение.

**Подход:** читать фактический HTML/DOM страницы после Save и определять признак успешного сохранения (toast, смена состояния кнопки, изменение элементов). Конкретный признак определяется при разработке по реальной странице YouTube Studio.

### Уникальный идентификатор видео = его точное название

Порядковый ID строки в таблице динамический (меняется при фильтрации). Название видео — стабильный и уникальный идентификатор.

### In-memory state (processed_titles: set)

Хранит названия уже обработанных видео **только в рамках текущего запуска**. При каждом перезапуске скрипта — чистый старт. Файл state.json не используется.
