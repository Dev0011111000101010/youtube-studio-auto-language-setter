# Вспомнить всё — состояние на момент перезагрузки (2026-03-13)

## Что мы делали

Исследовали, почему Playwright не мог подключиться к Google Chrome через CDP (порт 9222).

## Что обнаружили

### Факт 1: Chrome запущен с флагами, но порт НЕ слушается

`netstat` по порту 9222 вернул пустой результат — порт не открыт.

При этом WMI показал, что главный процесс Chrome (PID 21248) запущен с флагами:
```
chrome.exe --remote-debugging-port=9222 --remote-allow-origins=*
```
...а дочерние renderer-процессы также имеют этот флаг в командной строке.

### Причина: две проблемы одновременно

**Причина 1 — Single Instance (главная):**
Chrome на Windows — одноэкземплярное приложение. Если Chrome уже работал в момент запуска с `--remote-debugging-port=9222`, новый запуск просто передал флаги через IPC уже работающему процессу → тот их **проигнорировал**. Renderer-процессы унаследовали флаг от командной строки запуска, но главный browser-процесс (который и открывает порт) стартовал раньше — без этого флага. Поэтому `bind()` на 9222 не произошёл никогда.

**Причина 2 — Chrome 136+ требует `--user-data-dir`:**
Начиная с Chrome 136 (у нас версия 146), флаг `--remote-debugging-port` игнорируется, если не указан `--user-data-dir` с **нестандартной** директорией профиля. Это намеренное ужесточение безопасности.

## Гипотеза для проверки после перезагрузки

После чистой перезагрузки (Chrome не запущен) — запустить Chrome так:

```cmd
"C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222 --remote-allow-origins=* --user-data-dir="C:\Temp\chrome-debug-profile"
```

Затем проверить:
```cmd
netstat -ano | findstr :9222
curl http://localhost:9222/json/version
```

Если порт открылся и `/json/version` вернул JSON — CDP работает, можно подключаться из Playwright через `connect_over_cdp`.

## Открытый вопрос (не решён до перезагрузки)

Зачем вообще нужен внешний Chrome вместо `launch_persistent_context`?

Текущий `main.py` использует `p.chromium.launch_persistent_context(user_data_dir="./browser_profile")` — это запускает **собственный Playwright Chromium** с отдельным профилем. Он не зависит от системного Chrome.

Если цель — использовать **системный Chrome** (например, чтобы переиспользовать уже залогиненный профиль пользователя), то нужно либо:
- Вариант А: запустить Chrome с флагами (см. выше) и подключиться через `p.chromium.connect_over_cdp("http://localhost:9222")`
- Вариант Б: продолжать использовать `launch_persistent_context` с `browser_profile/`, но залогиниться в нём один раз вручную

**Этот вопрос нужно уточнить у пользователя после перезагрузки.**
